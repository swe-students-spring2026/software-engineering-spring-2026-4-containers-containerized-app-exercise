Web App Setup and Run Instructions

Project:
Fridge Food Detector Web App

Overview:
This web app allows users to upload a fridge image, run food detection using the machine-learning-client subsystem, store detected items in MongoDB, and view/edit/delete the inventory on a dashboard.

Requirements:
- Python 3.10+ recommended
- MongoDB running locally on port 27017
- machine-learning-client virtual environment already set up and working

Directory:
This README applies to the web-app/ subdirectory only.

--------------------------------------------------
1. Create and activate virtual environment
--------------------------------------------------

python3 -m venv venv
source venv/bin/activate

--------------------------------------------------
2. Install dependencies
--------------------------------------------------

pip install -r requirements.txt

--------------------------------------------------
3. Configure environment variables
--------------------------------------------------

Create a .env file inside web-app/ with the following content:

MONGO_URI=mongodb://localhost:27017/
DB_NAME=fridge_app
SECRET_KEY=dev-secret-key

--------------------------------------------------
4. Start MongoDB
--------------------------------------------------

If MongoDB is not already running, start it with Docker:

docker run --name fridge-mongo -d -p 27017:27017 mongo

If the container already exists but is stopped:

docker start fridge-mongo

--------------------------------------------------
5. Make sure the ML subsystem is ready
--------------------------------------------------

The web app calls the Python interpreter inside:

machine-learning-client/venv/bin/python

So the machine-learning-client virtual environment must already exist and must have all required ML dependencies installed.

The ML script used is:

machine-learning-client/food_detection.py

--------------------------------------------------
6. Run the web app
--------------------------------------------------

From inside web-app/:

python app.py

Then open in browser:

http://127.0.0.1:5000

--------------------------------------------------
7. Main features currently supported
--------------------------------------------------

- Upload a fridge image
- Save uploaded image locally
- Run ML food detection
- Store detected items in MongoDB
- Show detected items on dashboard
- Edit detected item names
- Delete items from dashboard
- Deduplicate repeated food classes from the same uploaded image

--------------------------------------------------
8. Notes
--------------------------------------------------

- Uploaded files are stored in web-app/uploads/
- Runtime ML task files are stored in web-app/runtime/
- The dashboard only shows items where is_deleted is False
- For duplicate detections of the same food in one uploaded image, only the highest-confidence result is stored

--------------------------------------------------
9. Reset database for testing
--------------------------------------------------

To clear all inventory items and uploads:

python -c "from db import get_db; db = get_db(); db.inventory_items.delete_many({}); db.uploads.delete_many({}); print('cleared')"

--------------------------------------------------
10. Common issue
--------------------------------------------------

If ML detection fails with missing packages such as cv2, torch, or transformers,
make sure the machine-learning-client virtual environment is correctly set up and that
the web app is calling:

machine-learning-client/venv/bin/python
